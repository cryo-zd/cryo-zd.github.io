DSANN, Approximate Nearest Neighbor Search of Large Scale Vectors on Distributed Storage

Overview

   In this paper, we present a new graph-cluster hybrid indexing and search system which supports Distributed Storage Approximate Nearest Neighbor Search, called DSANN. DSANN can effectively index, store, search billion-scale vector database in distributed storage and gurantee the high availability of of index service. DSANN employs the concurrent index construction method to significantly reduces the complexity of index building. Then DSANN applies Point Aggregation Graph to leverage the structural information of graph to aggregate similar vectors, optimiing storage efficiency and improving query throughput via asynchronous I/O in distributed storage.

Details

DSANN Framework

Motivation

   From our perspective, we can improve the ANNS performance of distributed storage scenarios by cmobinding the advantages of graph-based and cluster-based ANNS methods. Graph-based trversal makes asynchronous I/O and computation possible and partition exploration is well adapted to huge vector retrival because of large fan-out. In summary, we aim to design a new ANNS index with high performance from the following aspects. (1) making I/O and computation decouple and balance, (2) supporting huge vector retrieval, (2)working on distributed-stirage scenarios, (4) employing numerous cost-effective machines for parallel index construction. Point(1) can be achieved by the traversal of graph while fecthing vectors from distributed storage via network I/O. For point(2)-(3), the idea of clustering is similar to compression and partition exploration is well-adapted to huge vector retrieval. Regarding Point(4), we can distribute the data across multiple machines to construct localized proximity graphs, followed by subsequent merging. Below we propose a new graph-based structure called Point Aggregation Graph, which aggregates multiple close vectors into a vertex of the proximity graph.

Overview

   The overall pipeline of Point Aggregation Graph(PAG) construction is illustrated in Figure 2. DASNN samples a small portion(10-25%) of data as aggregation points of the concurrent construction of the proximity graph. The rest data points called residual points are aggregated to the nearest aggregation points within a distance limit by the search Algorithm 1 on the graph. In addition, to improve the performance of index, we leverage the structural information of the graph to choose multiple reduandant points for residual points. And then DSANN keeps aggregation points and the proximity graph in memory while storing the residual points in the distributed storage for the separation of computation and storage.

Index Construction

   Graph-based and cluster-based ANNS methods benefit from their respective index structure. Graph-based methods provide high accuracy while cluster-based methods can be viewed as a form of compression. It is intuitive to combines the adavatages of them to design a high-quality index structure that is tailored for distributed storage.
   Consider a proximity graph G(V, E) constructed on a dataset XXX consisting of billions of data points or even more. While ANNS on G using Algorithm 1 can achieve SOTA performance, it requires substantial memory to keep the index for online search. Dynamic Representation Selection

   Theoretically, if the ANNS interface provided by the PG G could achieve perfect nearest neighbor retrival, the naive PAG could be regared as a PG constructed on the centroids generated by K-Means algorithm with zero iteration. However, K-Means typically requires multiple iterations to ensure the quality of clustering and relying on zero iterations fail to provide such gurantee. Furthermore, the straightforward aproach of assigning each residual point to the partition associated with nearest neighbor may yield highly imbalanced partitions due to the data skew. Such imbalance in clustering introduces two primary challenges:

  • Long Tail Effect: Imbalanced clsutering may result in partitions of two extreme sizes
  • Low Recall Rate: Significant partition imbalance indicates that centroids generated through random sampling may not adequately represent the underlying distribution of data, leading to considerable boundary problem.

   To address the challenge of imbalanced partitions, we introduce the Dynamic Representation Selection(DRS) strategy in this section, which is summarized in Algorithm 3. (1) DRS explictly constraints the capacity of each partition to prevent excessively large partitions. The sample rate ppp can be conceptually interpreted as the compression rate with 1p\frac{1}{p}p1​ indicating that the number of points each partition should ideally preserve. To handle data skew, we allow partitions to moderately exceed their ideal capcity. Specifically, we set a partition capacity limitation of λp,λ≥1\frac{\lambda}{p}, \lambda \ge 1pλ​,λ≥1. (2) Additionally, the distance limit obtained through distance sampling ensures that only residual points within this distance can be aggergated to the corresponding aggregation point, as illustrated in Figure 6(a). The aggregation point, along with it residual points, defines a sphere with center as described in Definition 4. For adatpive aggregation, we calculate the distances between each aggregation point and its neighbors in PG, then use the γ1\gamma_{1}γ1​ -th sorted distance as the radiusthreshold for aggregation. However, some aggregation points in PG lack sufficient neighgbors, making the y1y_{1}y1​ position distance meaningless. Therefore, we sort radii of all aggregation points and use the γ2\gamma_{2}γ2​ position radisu as the maximum threshold for every aggregation point. (3) Finally, residual point that cannot be aggregated into any existing aggregation points will be promoted as new aggregation points by inserting them into the PG, as shown in Figure 6(b). Through the dynamic promotion, we aim to capture the underlying distribution of dataset.

Graph-based Redundancy

   Aggregation and clustering share many similarities and offen suffer from the bounday problem. To alleviate the boundary problom, we propose a Graph-based Redundancy(GR) strategy that select redundant partitions of residual point by using the structural information of the graph, thereby avoiding overly dense and spase redundancy distributions. The core idea of the redundancy strategy is to serve the search process:

  • Nearest Neighbor Redundancy: By utilizing the ANNS interface provided by the PG, we identify the k nearest neighbors of the residual points, and then apply RNG-based rules to select redundant partitions represented by aggregation points.
  • Routing Path Redundancy: We ovserve that similar queries often follow the same routing problem. During the greedy search process of the PG, we track the routing path, and then select redundant partitions along the routing path based on RNG rules.

Concurrent Index Construction

   Although sampling methods can significantly reduce the number of points required for constructing a PG, construting such a graph remains conputatinally expensive.
   In this section, we propose Concurrent Index Construction(CIC) to achieve efficient PG construction. The details are summaized in Algorithm 4. Following DiskANN methodology, CIC splits the dataset into multiple partitions for independent graph construction and then merges all individual graphs to a complete graph. Instead of merging the neighbors redundant points used by DiskANN, CIC let each partition’s point identify their nearest neighbors from the close partitions using ANNS interfave provide by the corresponding PG, and then employ prune strategy to preserve the properties of PG such as KNNG and RNG. More importantly, CIC can be easily scaled for distributed construction across multiple machines.
   Nevertheless, the cost of ANNS on all partition is considerable, we durther enhance CIC efficieny by limiting the number of partitions that each vector searches. Ideally, an effective clustering algorithm should constrain proximate points within the same partition and points residing in adjacent partitions are likely to exhibit greater similarity. Therefore, during the graph merge stage, each point needs only to consider its neighboring partitions. Our experimental results indicate that, generally, searching partitions within a distance threshld is sufficicent to achieve comparable effectiveness to searching all partitions. Formally, this can be expresses as δ(x⃗,xc′⃗)≤ηδ(x⃗,xc⃗)\delta(\vec{x}, \vec{x_{c^{'}}}) \le \eta \delta(\vec{x}, \vec{x_{c}})δ(x,xc′​​)≤ηδ(x,xc​​), where xc⃗\vec{x_{c}}xc​​ denotes the centroid of the partition containing x⃗\vec{x}x, xc′⃗\vec{x_{c^{'}}}xc′​​ represents the partition under consideration, and η\etaη is the scaling factor.

Search On Index

Query on Index

   During the search process, DSANN follows the inverted methodology. Initially, it traverses the PG stored in memory to identify the most suitable partitions. Then a find-grained full-scan within the partitions is performed to retrieve the final results.
   The number of partitions identified duting the graph traversal significantly influsences both the recall rate and througput. Employing a strategy that probes a fiexed number of partitions may lead to over-exploration of centain points. In this paper, we propose an Adaptive Partition Probe(APP) method, which dynamically explores partitions based on the structural information of the graph to achieve eraly stopping.
   During the traversal of the PG, DSANN maintains the distance information of each visited point, creating a timeline. The traversal terminated when the distance between the query and the current point exceeds d+ri+rjd + r_{i} + r_{j}d+ri​+rj​, where ddd represents the minimal distance to the centroid, rir_{i}ri​ denoted the aggregation radius of the nearest partition, and rjr_{j}rj​ is the aggregation radius of the current partition.

Asynchronous Query

   The inverted index method consists of two stages: a coarse-grained search followed by a fine-grained search. In the coarse-grained search stage, all residual points must be loaded into memory before entering the fine-grained stage. Therefore, the coarse-grained sage will stall the latter stage.
   In this paper, we introduce a search algorithm with asynchronous I/O and computation, as summarized in Algorithm 5. Leveraging the structural information of PG, DSANN can achieve asynchronous graph traversal in memory while concurrently retrieving the qualified partitions into mmeory for full-scan. By controlling the strictness of fetching criteria, we can effectively balance I/O and computation, improving the throughput of algorithm. For example, we can fetch the residual points as soon as the aggregation point with minimal distance is probed.
