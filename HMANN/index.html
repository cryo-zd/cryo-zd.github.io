HMANN, Efficient Billion-Point Nearest Neighbor Search on Heterogeneous Memory

Overview

  We present a new graph-based indexing and search algorithm called HM-ANN, which maps the hierarchical design of the graph-based ANNs with memory heterogeneity in HM, which takes both memory and data heterogeneity into consideration and enables billion-scale similarity search on a single node without using compression.(HM combines cheap, slow but extremely large memory with expensive fast but small memory to achieve a good balance between production cost, memory performance and capacity)
  Controbutions:

  1. Present a fast and accurate billion-scale nearest neighbor search solution on a single node without compression
  2. Exprole memory management techniques such as dynamic migration and parallel search
  3. Introduce a performance model to select search-related hyerparameters that satisfy search time and recall constraints

Details

  The design of HM-ANN generalizes HNSW. Elements in upper layer consume a small portion of the memory, making them good candidates to be placed in fast memory(small capacity), the bottom-most layer has all the elements and has the largest memory consumption, which makes it suitable to placed in slow memory. The key idea of HM-ANN is to build high-quality upper layers and make most memory access happen in fast memory, in order to provide bwtter navigation for search at L0L0L0 and reduce memory access in slow memory.
