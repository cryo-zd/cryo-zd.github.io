UNIFY, Unified Index for Range Filtered ANNS

Summary

   This paper presents an efficient and scalable framework for Range Filtered Approximate Nearest Neighbors Search(RF-ANNS) over high-dimensional vectors associated with attribute values. Though bilding dedicated indexes for each strategy(pre-filter/post-filter/hyrbid-filter) and selecting the best one based on qeury range can address this problem, it leads to index consistency and maintenance issues.
   Our framework, called UNIFY, constructs a unified Proximity Graph-based index that seamlessly suuports all three strategies. In UNIFY, we introduce SIG, a novel Segmented Inclusive Graph, which segments the dataset by attribute values. It ensures the PG objects from any segment combinations is a sub-graph of SIG, thereby enabling efficient hybrid filtering by reconstructing and searching a PG from relevant segments. Moreover, we present Hirarchical Segmented Inclusive Graph(HSIG), a variant of SIG that incoporates a hierarchical structure inspired by HNSW to achieve logarithmic hybrid filtering complexity. We also implement pre- and post-filtering for HSIG by fusing skip list connections and compressed HNSW edges into the hierarchical graph.

Segmented Inclusive Graph

SIG overview

   Attribute Segmentation. Given a dataset of nnn objects, RF-ANNS is concerend only with the n′n^{'}n′ objects within the query range, where n′<nn^{'} \lt nn′<n. To efficiently filter out the qualified objects, we employ the attribute segmentation method to reduce the search space. Assuming that the attribute distribution remains stable, we first sample objects from the dataset and sort them by their attribute values, then apply an equi-depth histogram to partition them. The histogram bin boudaries define the attribute intervals for each segment, partitioning the dataset into disjoint subsets with similar sizes for further indexing.
   Graph Inclusivity and SIG. After attribute segmentation, we focus on designing a graph for efficient hybrid filtering. Since a query range intersects a few segments, a straightforward approach is to independently build PGs for each segment. For an RF-ANNS query, we search the local PGs in intersected segments and merge the reuslts for the global KNN. However, this method results in search time increasing linearly with the number of intersected segments. We tackle it by introducing a novel graph family known as the Segmented Inclusive Graph(SIG). An SIG ensures that the PG of any segment combination is included in the SIG. Leveraging this characteristic, termed graph inclusivity, allows efficient hybrid filtering by reconstructing and searching smaller PG of a few segments covering the query range. Below, we introduce the formal definitions of graph inclusivity and SIG.

Hierarchical Segmented Inclusive Graph

   SIG-KNNG offers an efficient approach to build an SIG with segmented adjacency lists. Though it theoretically guatantees inclusivity, the underlying KNNG is not competitive with SOTA PGs like HNSW. Additionally, SIG-KNNG lacks support for incremental insertion. In this section, we expand the basic idea of SIG-KNNG, introducing the Hierarchical Segmented Inclusive Graph(HSIG), which uses HNSW as a building block to achieve incremental constrction and a logarithmic search complexity.
   Figure 4 provides an overview of HSIG, showcasing a hierarchical structure that follows the structure of HNSW and skip lists. HSIG is a unified structure that indexes both vectors and attributes by leveraging the strengths of vector-oriented HNSW and attribute-oriented skip lists. For vector indexing, HSIG organizes the outgoing edges of each object into chunks based on attribute segmentation, inspired by SIG-KNNG. Each chunk contains the edges of an HNSW for the corresponding segmentation. Thus, the backbone graph of HSIG can be seen as a set of HNSWs constructed for different segments, which are mutually connected with additional edges to ensure inclusivity. Additionally, we incoporate the skip list into the backbone graph to index attributes for efficient pre-filtering and introduce a compact auxiliary structure to optimize post-filtering.

Search on HSIG

   To adapt to different query ranges, we propose three search strategies and a range-aware search strategy selection method.
   Strategy A(Pre-Filtering). Navigated by the hierarchical skip list connections, we can quickly reach the bottom layer to identify the first object whose attribute value fall within the query range. From there, a linear search is performed to collect the query vector’s KNN among those vectors with qualified values.
   Strategy B(Post-Filtering). We employ the hierarchical search scheme in Algorithm 2 for ANNS, followed by filtering objects within the query range. ANNS is performed over a global HNSW with edges marked by bitmaps. During the search, only outgoing edges marked as 1 in an object’s bitmap are considered. The search retrieves the top-ef nearest neighbors(ef>kef \gt kef>k) in the bottom layer, after which attribute filtering is applied to obtain the top-k results.
   Strategy C(Hybrid Filtering). We perform hybrid filtering by reconstructing and searching the HNSW of segments covering the query range. The search follows the hierarchical scheme is Algorithm 2, starting from the topmost entry point of graphs in all segments. Since each object has MMM connections per segment, and assuming there are S′S^{'}S′ segments intersecting the query range, this requires visiting MS′MS^{'}MS′ neighboring nodes, leading to a large search space and high computational complexity. Thus, we introduce the search parameter mmm to reconstruct an HNSW with a maximum degree of mmm runtime. We propose two neighbor selection strategies to select mmm neighbors from MS′MS^{'}MS′ connections: (1) compute the distances between node vvv and its neighbors across all S′S^{'}S′ segments, then select the top-m neighbors by sorting them by distance; and (2) select the top-(⌈m/S′⌉)\text{top-}\big( \lceil m / S' \rceil \big)top-(⌈m/S′⌉) neighbors from each chunk of adjacency list. Since neighbors in each chunk are naturally ordered by distance to vvv upon acquisition vis ANNSearch, we simply take the first ⌈m/S′⌉\lceil m / S' \rceil⌈m/S′⌉ objects. We adopt the second strategy as the default.

   Range-aware Strategy Selection. Let YYY be the cardinality of objects within a query range. Observing the pre-, post-, and hybrid filtering perfrom best for small-, large, and mid-range queries. We propose the following heuristic for strategy slection: use strategy A if Y≤τAY \le \tau_{A}Y≤τA​, Strategy B if Y≥τBY \ge \tau_{B}Y≥τB​, and Strategy C if τA<Y<τB\tau_{A} \lt Y \lt \tau_{B}τA​<Y<τB​. Here, τA\tau_{A}τA​ and τB\tau_{B}τB​ are thresholds distinguishing the optimal ranges for each strategy, derived from historical data analysis. Given these thresholds, we estimate the cardinality of an incoming query to apply the heuristic. Since stattistic collection and cardinality estimation are well studied and not the focus of this paper, we provide a simple processing method to validate our heuristic. Specifically, we sample objects from the base dataset as queries and assign each a random query range. We then execute these queries using the three strategies and record recall and latency metrics. Given a recall target, we analyze records meeting this requirement and identify two turning points where pre- and post-filtering outperform hybird filtering. These points establish τA\tau_{A}τA​ and τB\tau_{B}τB​, guiding strategy selection for future queries.
