LSM-VEC, A Large-Scale Disk-Based System for Dynamic Vector Search

Overview

  This paper presents LSM-VEC, a disk-based dyanmic vector index that integrates hierarchical graph indexing with LSM-tree storage. By distributing the proximity graph across multiple LSM-tree levels, LSM-VEC supports out-of-place vector updates. It enhances search efficiency via a sampling-based probabilistic search strategy with adaptive neighbor selection, and connectivity-aware graph reordering further reduces I/O without requiring global reconstruction.
  (LSM-VEC is the first system to integrate the LSM tree to support efficient insertions and deletions in vector index.)

Details

Overview

  Fig2 presents the overview of LSM-VEC that is constructed upon a graph-oriented log-structured merge tree(LSM). Building on this foundation, we have integrated three key modules further to enhance the performance of LSM-VEC, each tailored to address specific challenges associated with disk-based ANN searches and updates:

  • LSM-based Hierarchical Graph Indexing Module: this module employs memory-disk hybrid hierarchical proximity graph that partitions the graph into memory-resident upper layers and a disk-resident bottom layer managed through an LSM-tree. The upper layer facilitate rapid long-range navigation, which the lower layers leverage efficient disk indexing and management. Vetcor storage and graph indexing are decoupled to enchance storage efficiency and enable quick disk-based vector retrival.
  • Sampling-Based Query Engine: this module implements a probabilistic neighbor selection mechanism based on projection-based similarity scores to selectively evaluate neighbors.
  • Connectivity-Aware Reordering Module: this module continuously optiimzes the layout of data based on observed access patterns to optimize random disk access. Unlike traditional static reordering methods, it dynamically leverages runtime traversal statics derived from the sampling-based query engine. Nodes frequently traversaled together are incrementally co-lacated during regular LSM-tree compactions, enhancing data locality and reducing random disk I/O.

LSM-based Proximity Embedding: Efficient Indexing for Dynamic ANN Search

  The original HNSW design assumes that the entire graph structure resides in memory, making it unsuitable for large-scale disk-based scenarios. To overcome this limitation, LSM-VEC decomposes the HNSW index into memory-resident upper layers and a disk-resident bottom layer. As shown in Fig2, the upper layers of HNSW are retained in RAM to support low-latency search entry and fast hierarchical navigation. According to the exponential decay distribution used in HNSW’s level assignment, the upper layers are typically small(Empirically less than 1% of all nodes reside above the bottom layer). Whereas the major layer of HNSW is stored on disk and maintained via an LSM-tree, facilitating efficient index updates. Since each vector insertion or deletion generates substantial new edges in the major layer, the adoption of an LSM-tree alllows LSM-VEC to handle these updates efficiently without requiring a global restructuring of the entire index. In additional, LSM-VEC stores vector data separatly from the graph index. All vectors are placed in a contiguous on-disk array, sorted by their corresponding ID. This layout allows constant-time retrival via offset computation, avoiding redundant data storage while ensuring that vector access and neighbor traversal remain both efficient and write-friendly.
  Search in LSM-VEC follows a layered traversal strategy, optimized to minimize random disk I/O. The search process starts from the upper memory-resident layers, where long-range edges enable efficient navigation towards the target region. Once the search reaches the lower disk-resident layer, LSM-VEC employs the sampling-guided traversal technique to selectively explore a small set of promising neighbors, which significantly reduces unnecessary disk accesses.
  Insertion in LMS-VEC follows a hierarchical HNSW-style process. The vector is assigned to a random level LLL sampled from an exponentially decaying distribution. The insertion then proceeds top-down through the hierarchy: at each layer l\mathcal{l}l(except the bottom layer), the system identifies approximate neighbors and connects the vector to the top-M closet nodes using in-memory search. At the bottom layer, neighbor search is conducted on the disk-resident graph stored in the LSM-tree. The vector is connected to the top-M nearest disk-resident nodes, and the resulting edges are written to the LSM-tree for durable strage.

  Deletion in LSM-VEC performs a local neighbor relinking strategy for both in-memory and disk-resident layers. When a vector is deleted, its immediate neighbors are reconnected using approximate neighbor search to preserve local graph connectivity. For the disk layer, LSM-VEC identifies affected nodes and inserts new edges into AsterDB, avoiding full reindexing. After relinking neighbors, the system removes all edges involving the deleted node from AsterDB and deletes the corresponding vector data.

Sampling-Guided Traversal: Fast and Robust Search over Disk-Based Graphs
