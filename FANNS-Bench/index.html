FANNS-Bench, A Unified Benchmark and Systematic Experimental Study

Overview

   A comprehensive survey and benchmark for FANNS is crucial to achieve the following goals: designing a fair evaluation and clarifying the classification of algorithms, conducting in-depth analysis of their performance, and establishing a unified benchmark. First, we propose a taxonomy(dividing methods into filter-then-search, search-then-filter, hybrid-search) and a systemmatic evaluation framework, integrating unified parameter tuning and standardized filtering across algorithms to reduce implementation-inducd performace variations and reflect core trade-offs. Then, we conduct a comprehensive empirical study to analyze how query difficulty and dataset properties impact performance, evaluating robustness under pressure like filter selectivity, Recall@k, and scalability to clarify each method’s strengths. Finally, we establish a standardized benchmark with real-world datasets and open-source related resources to ensure reproducible future reasearch.

Filter-Then-Search ANNS Algorithms

   Filter-then-search algorithms explicitly or implicitly filter the entire dataset D={x1,x2,...,xn}D = \left \{ x_{1}, x_{2}, ..., x_{n} \right \}D={x1​,x2​,...,xn​} to extract vectors satisfying specified label constraints. This process generates a filter subset Df={xk1,xk2,...,xkm}D_{f} = \left \{ x_{k1}, x_{k2}, ..., x_{km} \right \}Df​={xk1​,xk2​,...,xkm​} where Df⊆DD_{f} \subseteq DDf​⊆D. Subsequent nearest neighbor search operates exclusively within this filtered subset DfD_{f}Df​, requireing no additional constraint verification during query processing. The search procedure reduces to solving a conventional ANNS problem over the constrained dataset.
   A critical implementation combines pre-filtering with vector quantization. This approach first utilizes a fast bitset to identify the IDs of all points that satisfy the metadata constraints. Then, a quantization algorithm is used to rapidly compare the approximate distances between these candidate points and the query vector. Finally, a small subset of the most promising candidates is selected for a reranking step using their full-precision vector representations.
   Creating the filter map relies on a pre-computed inverted index, where each label maps to a bitset indicating all vectors that possess it.
   Pre-filter Brute Force. We implement a baseline algorithm that first filter the dataset and then applies brute-force search. The filter map is used to retrieve the subset of database (Df⊆D)(D_{f} \subseteq D)(Df​⊆D) that satisfies the filter constraints. Then, it performs a brute-force, full-precision search within filtered vector set(DfD_{f}Df​) and returns the top-k results.
   ACORN-γ. The ACORN-γ algorithm constructs its without any label information in advance. This follows for flexible data updates, as labels can be assigned or modified after the index is built. The core construction idea is to create a denser graph by expanding each node’s degree by a factor γ\gammaγ, which is dtermined based on an estimated selectivity. A specialized predicate-agnostic pruning techinuqe is then applied to control the index size. At query time, ACORN-γ performs a full-dataset scan to generate a bitset of all vectors satisfying the filter constraint(which can be complex as a regular expression). The subsequent graph traversal is then guided by this bitset. When expanding its candidate list, it decompresses the neighbor information and checks filtering conditions, which broadens the search’s scope. This algorithm can be combines with mainstream indexes like HNSW and DiskANN and performs wel with high selectivity, and many vector search engines have added support for it.
   ACORN-1. In contrast to ACORN-γ, ACORN-1 offers a lightweight alternative that aims to approximate the search performance of ACORN-γ while significantly reducing index construction time and memory cost. The key difference lies in its approach to neighbor expansion. Instead of creating a denser graph during construction, ACORN-1 builds a standard, un-pruned HNSW index. During the greedy search, then visiting a node vvv, ACORN-1 dynamically expands the search scope to consider not only its direct(one-hop) neighbors but also its two-jop neighbors. This expanded candidate set is then filtered according to the query predicate before the best neighbors into the priority queue.
   UNG. The UNG algorithm precompiles lable information into a label navigating graph structure during offline construction, which encodes the containment relationships of different label sets. It groups vectors by their labels, builds intra-group graphs, and adds inter-group edges based on a label navigating graph(LNG). This design gurantees that if an edge (vi,vj)(v_{i}, v_{j})(vi​,vj​) exists, their labels satisfy a containment relationship (fi⊆fj)(f_{i} \subseteq f_{j})(fi​⊆fj​). By locating the entry labels that follow the filter constraints, greedy traversal on the proximity graph never requires further label checking, making UNG a highly efficient filter-then-search method for containment and equality predicates. However, its performance suffers significantly on queries with more complex constraints, such as label overlap scenarios, as multiple rounds of search are performed starting from different entries.

Hybrid-Search ANNS Algorithm

   Hybrid-search algorithms integrate label constraints directly into the search process, avoiding explict pre-filtering or post-filtering stages. This approach typically employs some strategies, such as constraint neighbor expansion or distance fusion. They may build upon fundationaly ANNS methods, such as Filtered-DiskANN that extends DiskANN by deeply integrating label constraints into index construction and query processing.
   Filtered-DiskANN. This algorithm is based on the Vamana graph, the core graph index of DiskANN, but modifies both the construction and search process to be label-aware. During index construction, when a new point is inserted, only neighbors that share at least one common lable with the new point(i.e., fi∩fq≠∅f_{i} \cap f_{q} \neq \emptysetfi​∩fq​=∅) are added to the candidate queue, and edges are then established with the candidates found along this search path. Finally, a lable-aware pruning strategy is applied to refine the connections. We note that the original implemention primarily supports singleton query label sets (∣fq=1∣|f_{q} = 1|∣fq​=1∣). To accomodate more genral cases (∣fq>1∣|f_{q} \gt 1|∣fq​>1∣) and more scenarios, we extend the algorithm by adding an extra filtering step: after a node is poped from the candidate queue and before it is placed into the final result, we perform a label check.
   Stitched-DiskANN. In contrast to Filtered-DiskANN’s graph approach, Stiched-DiskANN adoptes a “partition-then-stich” construcion strategy. It first partitions the dataset by creating a separate Vamana subgraph for each individual label in the entir label set L\mathcal{L}L. During the construction of each Vamana subgraph, a point is connected to other points whose labels intersect with its own; this design makes its search process highly suitable, as it only needs to expand neighbors that have label intersections with the query and ensures connectivity. These intially disconnected subgraphs are then “stiched” together into a single cohesive index. The stiching process leverages points that carry multiple labels, sinces these points naturally appear in multiple subgraphs, they serve as crucial bridge nodes that ensure connnectivity.
   NHQ. NHQ is a hybrid-search algorithm that builds upon fundationaly graph-based ANNS methods, such as kgraph and NSW, and introduces modifications to better adapt to filtering scenarios. It integrates label constraints into both the graph construction and query process by fusing label-based distances with vector distance metrics. Specifically, NHQ defines a combined weight wt(vi,vj)wt(v_{i},v_{j})wt(vi​,vj​) using the Hamming distance of the labels between two nodes and the vector distance δ(vi,vj)\delta(v_{i}, v_{j})δ(vi​,vj​), creating a weighted fusion of these two metrics. For graph construcion, NHQ employs a Navigable PG(NPG) strategy, where edges are proactively established between points in diverse directions. This ensures better navigability during the search process, allowing queries to traversa the graph efficiently.
   CAPS. CAPS algorithm construts a two-level inex by first partitioning the dataset into coarse spatical clusters using k-means. Each cluster is then further subdivided into a hierarchy of fine-grained, label-based sub-clusters. This is done by iteratively creating sub-clusters for the most frequent label among the remaining vectors, up to a specified number of sub-clusters hhh. At query time, CAPS first identifies the nearest coarse clsuter centroid. It then sequentially probes the second-level sub-clusters within that cluster in their order of creation to find matching results.

Parameter Tuning

   Existing studies demonstrates that vector databses exhibit a high dimensional parameter space where configurations are highly interdependent optimization of individual parameters. Fuethermore, tuning constitutes a complex two-objective optimization problem(maximizing both QPS and recall), making it challenging to identify parameter configurations that yeild optimal trade-offs in the QPS-Recall space.

  • Coverage: Comprehensive explotation of the parameter space
  • Balance: Optimal trade-offs vis the Pareto frontier
  • Robustness: Near-optimal paramters for each specific context

For each parameter configuration θ\thetaθ within a subspace, we evaluate its performance across all defined search scenarios and compute the average QPS and Recall(line 11).
   A key step in our method is to normalize the comparison of different parameter sets. We evaluate how efficiently a confiuration can achieve specific recall targets. We define a set of fixed recall levels(e.g., 0.8, 0.9, 0.95) (linre 15) and use linear interpolation to estimate the QPS that each configuration θ\thetaθ would achieve at these exact recall values(line 13). Each configuration is then assigned a rank based on its interpolated QPS values(line 14). Finally, for each subspace, we select the parameter configuration with the best overall rank as the representative of that region

Recommendation

(We give conclusions directly below, refer the original paper for more experiment details)

  • Prefer UNG default for strong QPS-recall in Containment and Equality scenarios; uder very low selectivity or long labels, filter-then-search(e.g., UNG, ACORN-1, or even pre-filter Brute Force) keeps candidates small and throughput high; at high k, favor UNG/ACORN and avoid post-filter IVFPQ
  • Fixed-Length Equality is an easier, structured case where most methods reach high recall with proper tuning; UNG and CAPS offer top QPS at high recall
  • Prefer ACORN-1 and stitch-diskann in overlap scenario for balanced thoughput and recall; ACORN and post-filter HNSW are robust for large label space or high k; avoid UNG as query becoms more complex causing QPS/recall degradation
  • At low k(e.g., recall@1), post-filter methods and UNG are competitive with QPS; at higher k(e.g., recall@100), prefer UNG/ACORN for containment/equality and DiskANN/ACORN for overlap
  • If indexing time or memory is constrained, HNSW, IVFPQ, and ACORN-1 build quickly, are more memory-efficient, and offer greater flexibility as base labels change. Methods that rely on predicate label distribution(e.g., UNG, DiskANN variant) may incur higher build costs.
