RWalks, Ramdom Walks as Attribute Diffusers for Filtered Vector Search

Summary

   Existing solutions for filtered vector search are either: (1)ad-hoc, using existing techniques with no or minor modifications; (2)hybrid, providing specialized indexing and/or search algorithms. We xhow that neither is satisfactory and propose RWalks, an index-agnostic graph-based filtered vector search method that efficiently supports both filtered and unfiltered vector search. The key design features of RWalks include:

  1. A hybrid distance metric
  2. An index-agnostic attribute enrichment process
  3. A distance pruning heuristic

RWalks

   We propose three key desirable features of Filtered Vector Search solutions for efficiency and robustness: (1) the filter specificity should be decoupled from the recall-latency trade-off to ensure the method is robust in query workloads or varied specificity; (2) the index should be constructed to support both filtered and unfiltered vector search with minimal impact on index maintenance; (3) both index construction and search should scale well with large datasets, high dimensional, and different filter types.
   RWalks support both filtered and unfiltered vector search thanks to: (1) a post-processing algorithm that can enrich any existing graph-based vector index with attribute information(without altering the original index), and (2) a query-answering algorithm that modifies the greedy search used by state-of-the-art approaches to support filtered vector search.[ RWalks is index-agnostic, therefore, it can be used with any graph-based vector search method ]

Relaxing Specifity/Latency Dependency

   Query answering using HNSW in the unfiltered case consists of traversing a graph using a greedy search algorithm and populating a priority queue (W)(W)(W) with potential neighbors. The size of queue acts as termination condition, controlling the search latency. To support filtered search, HNSW-inline modifies the criterion for adding candidates to WWW: instead of adding candidates based on their distance to query only, it adds them only if they satisfy the query filters. This ensures recall, as all points aggregated in WWW satisfy the query filter. However, it can slow down the graph traversal in low-specificity workloads since filling up WWW takes a longer time. Removing this controlled addition of candidates to WWW results in stable traversal length regardless of specificity, but may affect recall guarantees a not all points in WWW validate the filteres. That is why solving for latency affects recall. To alleviate the issue, we build upon the following observation:
   Observation 1. Given the ounded size of WWW, points in the queue are poped when better candidates are found(i.e., closer to the query). Yet, these poped elements may satisfy the query filteres and should not be discarded.
   We introduce a twin priority queue (Wt)(W_{t})(Wt​) where poped elements that meet the query filteres are added. Once the search is complete, WtW_{t}Wt​ is merged with WWW. This simple modification reduces the drop in recall compared to a baseline HNSW search while stabilizing latency, as we eleminate the controlled addition of points to WWW thus keeping the traversal length same as in an unfiltered search.

Improving Recall with a Biased Traversal

   The proposed search modification based on Observation 1 maintains stable latency across various filter regimes, shifting the impact of filter specificity from latency to recall, albeit not to the same extent as a baseline HNSW search. For instance, with a uniform distribution of attributes, the ratio of valid points encountered during graph traversal will mirror the filter specificity. Therefore, lower specificity leads to fewer valid points, resulting in fewer candidate neighbors and consequently, lower recall. This effect is amplified if the attribute distribution negatively correlates with the vectors.
   We address this limitation by proposing a biased graph traversal that prioritizes visiting valid points. We achieve this by using a hybrid distance metric during search, unlike HQANN that uses a hybrid distance during both index construction and search.

Relaxing Sparsity with Graph Diffusion

   In low-specificity regimes, particularly there is no clustering(points in TTT are uniformly distributed in the graph), this hybrid distance may become ineffectiv. As explained in our second observation below.
   Observation 2. In low-specificity wrokloads with no clustering, few nodes share the same attributes as the query, i.e., the probability of a traversed node beling valid is low(attribute sparsity), rendering obsolete the attribute component of the hybrid distance.
   During graph traversal, neighbors of visited nodes are evalutaed using a binary question: does this neighbor satisfy query filters? We enrich the node’s attribute representation to answer the additional question: would this neighbor guide us toward nodes that satisfy query filters? We achieve this with a novel post-indexing attribute enrichment process(Algo. 1) that transforms the attribute representation from a binary form that encodes only the attributes of a given node, to a dense representation that is aware of the attributes of a node and its surroundings. We exploit graph diffusion, inspired by lable propagation in node classification where labeled nodes propogate their labels to their unlabeled neighbors.
   D-hop Random Walks as Attribute Diffusers. RWalks creates a rich attribute representation for each node $V_{i} by running multiple random walks starting from ViV_{i}Vi​, on an existing graph, and aggregates the attributes of all visited nodes via an average.

Pruning Distances with Attribute Heuristics

   In RWalks, the neighbors of a visited node can be partitioned into three classes: (1)neighbors that satisfy the query filter; (2) neighbors that do not satisfy the query filters but might guide the traversal toward valid regions in the graph; (3) neighbors that do not satisfy the query filter nor guide search toward valid regions. To improve search efficiency, we propose the following heuristic: during search, only nodes that belong to the first two classes are visited. The choice of neighbors to be evaluated is done by z-normalizing the attribute vectors in the indexing process, then applying during search a user-defined threshold τ≥0\tau \ge 0τ≥0 to decide which neigibors to visit. Future

  • This paper propose an index-agnostic framework for graph-based ng-approximate filtered vector search using the Euclidean distance. Support for other distance measures, types of search(e.g., range queries) and theoretical guarantees on query accuracy are out-of-scope and part of our future work
