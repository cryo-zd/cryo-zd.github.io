ESG, Elastic Graphs for Range-Filtering Approximate k-Nearest Neighbor Search

Overview

  Range-filtering approximate k-nearest neighbor(RFAKNN) search takes as input a vector and a numeric value, returing k points from a database of N high-dimensional points. The returned poins must lie within the specified query range, and they must be approximately the k nearest points to the query vector. A straightforward approach to processing RFAKNN queries is to filter out points outside the query range and then perform a search on the remaining in-range points. While indexing can speed up this process, ensuring correctness would require indexing all possible O(N2)O(N^2)O(N2) query ranges, which is infeasible due to severe storage constrains. To address these challenges, existing methods index only a subset of ranges, organized within a segment tree. At query time, the query range is decomposed into multiple subranges stored in the segment tree, but they also introduce significant query overhead since processing a single query requires combining indexes from O(logN)O(log N)O(logN) subranges.[ 面向标量范围查询场景，线段树，分而治之+归并/重建图索引 ]
  In this paper, we introduce a paradigm from the commonly used Prefiltering principle to the PostFiltering principle and propose novel reconstruction-based methods that guarantee at most two subranges are required for a query.

   For Graph Index, there are two principles for answering RFAKNN search queryies:

  • PreFiltering: identify and diacard out-of-range points during the search process. It is effective in removing out-of-range points from the graph and reducing the search space. As a result, it ensures high query efficiency but may cause the graph to become sparse, potentially preventing the search from finding the correct result.
  • PostFiltering: do not discard out0of-range points during the search phase, only when a point is encountered and evaluated for inclusion in the result set, it discards the points if its numericvalue is not within the quey range. It maintains high accuracy because it searches on the whole graph, but it may result in a longer query time.

  Recall that PreFiltering and PostFiltering only work on single graph index created for the entire range [1,N][1,N][1,N], and to overcome their limitions, existing solutions primarily focus on creating multiple graph indexes, and these solutions can be categoriced into two types:

  • Compression-based methods: construct the graph index for each of the O(N2)O(N^2)O(N2) possible ranges. When a RFAKNN query is issued, the graph corresponding to the query range [lq,rq][l_q,r_q][lq​,rq​] is retrived, and the query is processed using a graph search algorithm. Since materializing and storing all O(N2)O(N^2)O(N2) possible ranges is infeasible, thses methods propose a holistic index that compactly compresses these O(N2)O(N^2)O(N2) indexes to a size of O(NlogN)O(NlogN)O(NlogN).[ 压缩意味着信息丢失和精度下降 ]
  • Reconstruction-based methods: instead of materializing the index for all possible ranges, a subset of ranges is selected to index. Specifically, a segment tree is employed for this selection purpose. Initially, the entire range [1,N][1,N][1,N] is selected as the root and indexed. Subsequently, the range is divided into sub-ranges(i.e., 2 for fanout) and this division continues until the number of points within a given subrange falls below a predifined threshold(O(NlogN)totalindexesO(NlogN) total indexesO(NlogN)totalindexes). When a query with a range [l,r][l,r][l,r] is issued, the segment tree can reconstruct the index(combined to generate the index for query range)[ 因为需要合并若干子索引来重建索引，性能有一定下降 ]

RFAKNN Query for General Queries

  For handling a general RFAKNN query, we propose ESG2DESG_{2D}ESG2D​ index, which is also a hierarchical structure encoded in a segment tree.   To process a query with qqq and a range [lq,rq][l_q,r_q][lq​,rq​], we follows a recursive process, with the termination condition being when the selected subrange [l,r][l,r][l,r] becoms samller than a predefined threshold. At this point, a linear scan is performed to report the answer. If the query range is ccc time larger than the selected subrange [l,r][l,r][l,r](whre c is the elastic factor), the graph stored in ESG2DESG_{2D}ESG2D​ is utilized along with PostFiltering to find the answer(Line 3-5). Otherwise, the subrange is divided into two parts, and the algorithm is applied recursively to each part: the left part and the right part. The results from the left and right parts are merged to produce the final answer. A key difference is that when a subrange encountered during traversal does not exactly match the current query range, it still can be used if Line 3 satisfies, thanks to the use of the elastic factor.
