AllanPoe, All-in-one Graph-based Indexing for Hybrid Search on GPUs

Overview

   This paper presents Allan Poe, a groundbreaking unified graph index that fundamentally transforms hybrid search by integrating four retrieval paths—dense vector, sparse vector, full-text, and knowledge graph—within a single GPU-accelerated framework. The system addresses the critical limitations of existing paradigms through an innovative design featuring isolated heterogeneous edge storage and dynamic path fusion, achieving unprecedented performance in accuracy, efficiency, and flexibility simultaneously.

Details

   Allan Poe introduces a Unified Semantic Metric Space (USMS) that mathematically formalizes the integration of diverse retrieval methods. The core innovation lies in its isolated heterogeneous edge storage mechanism, which maintains distinct edge types while unifying them within one cohesive graph structure, enabling seamless combination of retrieval paths without reconstruction overhead.

Analysis

   The paper begins by diagnosing fundamental limitations in current retrieval ecosystems through comprehensive experimental analysis:

   THE TRILEMMA OF EXISTING PARADIGMS:

  • Single-path Retrieval (HNSW, BM25, SPLADE): Achieves high efficiency but suffers from inherent semantic gaps, with vector similarity failing to guarantee end-to-end document relevance (only ~50% accuracy in complex queries).
  • Multi-path Retrieval (Infinity, Milvus): Provides flexibility through separate-then-fuse strategies but incurs massive storage overhead (up to 21× larger indexes) and accuracy degradation due to suboptimal result fusion.
  • Fusion Retrieval (IVF-Fusion, DS-ANN): Improves accuracy over single-path methods but lacks flexibility, typically limited to two-path combinations and requiring complete reconstruction for weight changes.

   KEY MOTIVATIONAL FINDINGS:

  • Experiments on NaturalQuestions and 2WikiMultiHopQA reveal that while vector similarity accuracy reaches 99%, end-to-end accuracy remains substantially lower (around 50-70%), exposing the semantic gap limitation.
  • Hybrid search demonstrates clear advantages, but no single configuration dominates all scenarios—optimal path combinations vary significantly across datasets and query types.
  • The analysis concludes that an ideal solution must simultaneously deliver: Flexibility (arbitrary path/weight combinations), Accuracy (complementary multi-path information), and Efficiency (low-latency heterogeneous computation).

Keyword Edges Supplement

   To preserve exact keyword matching capabilities compromised by vector fusion, Allan Poe introduces a dual-assessment mechanism during graph pruning. The algorithm selectively preserves pruned edges as dedicated keyword edges based on keyword overlap conditions:

   For node AAA with pruned neighbor XXX, if ∄Y∈neighbors(A)\nexists Y \in \text{neighbors}(A)∄Y∈neighbors(A) such that K(A)∩K(X)⊆K(Y)K(A) \cap K(X) \subseteq K(Y)K(A)∩K(X)⊆K(Y), then XXX is recycled as a keyword edge. This ensures navigation paths for important keyword relationships remain accessible.

GPU-Accelerated Index Construction

   The four-stage construction pipeline leverages massive GPU parallelism for efficient hybrid index building:

   PROCEDURE 1: Initial k-NN Graph Construction

  • Employs NN-Descent algorithm with warp-level hybrid distance computation
  • Optimized dense vector processing using CUDA vectorized instructions
  • Sparse vector similarity transformed into parallel set intersection with shared memory caching

   PROCEDURE 2: RNG-IP Joint Pruning

  • Combines Relative Neighborhood Graph pruning with Inner Product pruning
  • Maintains search quality while reducing index complexity
  • Preserves aligned degree structure for GPU-friendly parallel processing

   PROCEDURE 3: Keyword-aware Neighbor Recycling

  • Recycles pruned edges as keyword edges based on keyword overlap conditions
  • Leverages computations from previous procedures to minimize overhead
  • Uses hash maps in GPU shared memory for efficient keyword lookups

   PROCEDURE 4: Logical Edge Augmentation

  • Maps knowledge graph entities to document nodes
  • Establishes logical connections between related documents
  • Optional component trading accuracy improvements against construction cost

Flexible Query Processing Framework

   The query processing engine supports dynamic path combinations through three sophisticated mechanisms:

   BASE SEMANTIC EDGES QUERY:

  • Dynamically fuses query vectors: fconcat(q,wd,ws,wf)=[wd∗fdense(q),ws∗fsparse(q),wf∗ffull(q)]f_{\text{concat}}(q,\mathbf{w}_{d},\mathbf{w}_{s},\mathbf{w}_{f})=[\mathbf{w}_{d}*f_{\text{dense}}(q),\mathbf{w}_{s}*f_{\text{sparse}}(q),\mathbf{w}_{f}*f_{\text{full}}(q)]fconcat​(q,wd​,ws​,wf​)=[wd​∗fdense​(q),ws​∗fsparse​(q),wf​∗ffull​(q)]
  • Single-path retrieval achieved by setting corresponding weight to 1 and others to 0
  • Optimized entry point selection from low-norm nodes

   DYNAMIC KEYWORD EDGE LOADING:

  • Implements just-in-time loading of keyword edges only for nodes sharing keywords with query
  • Employs twin candidate pool approach to maintain accuracy while filtering results
  • Ensures keyword constraints without compromising traversal pathways

   KNOWLEDGE GRAPH AUGMENTATION:

  • Uses entity-based entry point selection via entity-node mapping
  • Dynamically loads logical edges for nodes within xxx hops of query entities
  • Incorporates logical similarity through hop-distance-based reward mechanisms
  • Enables fine-grained entity-document joint traversal for complex queries
