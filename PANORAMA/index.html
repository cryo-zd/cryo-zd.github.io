PANORAMA, Fast-Track Nearest Neighbors

Overview

   Currently, ANNS systems spend up to 99% of query time to compute distance in their final refinement phase. In this paper, we present PANORAMA, a machine learning-driven approache that tackles the ANNS verification bottleneck through data-adaptive learned orthogonal transforms that facilitate the accretive refinement of distance bounds. Such transforms compact over 90% of signal energy into the first half of dimensions, enabling eraly candidate pruning with partial distance computations. Our contributions are as follows:

  • Cumulative distance computation: We introduce PANORAMA, an accretive ANNS refinement framework that complements existing ANNS schemes to render them effective on modern workloads. PANORAMA incrementally accumates L2 distance terms over an orthogonal transform and refines lower/upper bounds on the fly, promtly pruning candidates whose lower distance bound exceeds the running threshold.
  • Learned orthogonal transforms: We introduce a data-adaptive Cayley transform on the Stiefel manifold that concentrates energy in leading dimensions, enabling tight Cauchi-Schwarz distance bounds for early pruning.
  • Algorithm-systems co-design: We carefully co-design system aspects with specialized variants for contigunous and non-contigunous memory layouts, leveraging SIMD vectorization, cache-aware layouts, and batching, also provide theoretical guarantees alongside practical performance.
  • Integrability: We fold PANORAMA into five key ANNS indexes(IVFPQ, IVFFlat, HNSW, MRPT, Annoy) to gain speedups without loss of recall and showcase its efficaciousness through experimentation across datasets, hyperparameters, and out-of-distribution queries.

Learning Orthogonal Transforms

   We propose learning a tailored linear orthogonal transform for ANNS purpose. Formally, we seek a matrix T∈Rd×dT \in R^{d \times d}T∈Rd×d, with TTT=IT^{T}T = ITTT=I, such that the transform z=Txz = Txz=Tx of a signal x attains energy compaction, i.e., concentrates most energy in its leading dimensions while preserving norms by orthoganality, i.e., ∣∣z∣∣2=∣∣x∣∣2||z||_{2} = ||x||_{2}∣∣z∣∣2​=∣∣x∣∣2​

Parameterization

   We view the set of orthogonal matrices, O(d)={T∈Rd×d:TTT=I}O(d)=\left \{ T \in R^{d \times d}: T^{T}T = I \right \}O(d)={T∈Rd×d:TTT=I}, as the Stiefel manifold, a smooth Riemannian manifold where geodesics corresponds to contious rotations. The Cayley transform maps any d×dd \times dd×d real skew-symmetric(antisymmetric) matrix AAA —— i.e., an element of the Lie algebra so(d) of the special orthogonal group SO(d), with AT=−AA^{T}=-AAT=−A, hence having dim=d(d−1)/2dim=d(d-1)/2dim=d(d−1)/2 independent entries —— to an orthogonal matrix in SO(d) (exclusing rotations with -1 eigenvalus). The resulting matrix lies on a subset of the Stiefel manifold, and the mapping servers as a smooth retraction, proving a first-order approximation of a geodesci at its starting point while avoiding repeated projections:

T(A)=(I−γ2A)−1(I+γ2A)(5)T(\mathbf{A}) = \left( I - \frac{\gamma}{2} \mathbf{A} \right)^{-1} \left( I + \frac{\gamma}{2} \mathbf{A} \right) \tag{5} T(A)=(I−2γ​A)−1(I+2γ​A)(5)

   The parameter γ\gammaγ controls the step size of the rotation on the Stiefel manifold: smaller γ\gammaγ values yield smaller steps, while larger values allow more aggresive rotations but may risk numerical instability. Contrary to other parameterizations for orthogonal transform operators, such as updates via Householder reflections and Givens rotations, which apply a non-paralelizable sequence of simple rank-one or planar rotations, the Cayley map yields a full-matrix rotation in a single update step, enabling efficient learning on GPUs without ordering bias. Unlike structuredfast transforms, which rely on sparse, rigidly defined matrices crafted for special data types, the learned transform is dense and full determined by the data, naturally adapting to any dataset. Further, the Cayley map enables learning from a rich and continuous family of rotations; altoough it excludes rotations with -1 as an eigenvalue, which express a half-turn in some plane, it still allows gradient-based optimization using standard batched linear-algebra primitives, which confer numerical stability, parallelizability, and suitability for GPU acceleration.

Integration with SOTA Indexes

   On contiguous layouts, which exploit locality and SIMD parallelism, we rearrange the contiguous storage to a level-major format to facilitate PANORAMA’s level-wise refinement and bulk pruning in cache-efficient fasion. On non-contigunous layoouts, PANORAMA still curtails redundant distance computations, despite the poor locality. Here, we discuss how we integrate PANAROMA in the refinement step of both categories.

Contiguous-Layout Indexes

   L2Flat and IVFFLAT’s native storage laout does not suit PANORAMA for two reasons:

  1. Processor cache locality and prefetching: By PANORAMA refinement, we reload query slices for each each vector, preventing stride-based prefetching and causing frequent processor cache misses.
  2. Branch misprediction: While processing a single vector, the algorithm makes up to nlevelsn_{levels}nlevels​ decisions on whether to prune it, each introducing a branch, which rendes control flow irregular, defeats branch predictors, and stalls the instruction pipeline.

   To address these conserns, we integrate PANORAMA in Faiss with a batched, level-major design, restructuring each clsuter’s memory layout to support level-wise rather than vector-wise refinement. We group vectors into batches and organize each batch in level-major order that generalizes the dimension-major order that generalizes the dimension-major layout of PDX. Each level stores a contiguous group of features for each point in the batch. The algorithm refines distances level-by-level within each batch. At each level, it first computes the distance contributions for all vectors in the batch, and then makes bulk pruning decisions over all vetcors. This consolidation of branch checks in nlevelsn_{levels}nlevels​ synchronizes steps regularizes control flow, reduces branch mispredictions, and improves cache utilization.
   However, directly applying the storage layout of L2Flat/IVFFLAT to IVFPQ introduces an additional chalenge:
3. SIMD lane underutilization: When the PQ codes for a given vector are shoter than the SIMD register width, vector-wise leaves many lanes idle, underusing computr resources.

   Instead of storing PQ codes by vector, we contiguously store code slices of the same quantizer across vectors in a batch. This layout lets SIMD instructions process lookup-table(LUT) entries for multiple vectors within the register, fully utilizing compute lanes and address cache thrashing, as LUT entries of codes for the same query lines remain cache-resident for reuse.

Non-Contiguous-Layout Indexes

   Our index methods that store candidate points in noncontiguous memory, the refinement phase faces a memory-computation tradeoff. Fetching candidate vectors incurs frequent prcessor(L3) cache misses, so the cost of moving data into cache rivals that of arithmetric distance computations, rendering the process memory-layout. Even with SIMD acceleration, poor locality among candidates slows throughput, and by Amdahl’s lay, enhancing computing alone yoelds disminishing returns. Lacking a good fix, we do not rearrange the storage layout with these three indexes.
   Graph-based(HNSW). Unlike other ANNS methods, HNSW conducts no separate verifiction, as it computrs exact distances during traversal. We integrate PANORAMA by modifying how embedding enter the candidate heap to reduce distance evaluations: we prioritize candidates using running bounds, with the estimate LBl+UBl2\frac{LB^{l} + UB^{l}}{2}2LBl+UBl​, and defer computing a candidates’s exact distance until it enters the result heap.
   Tree-based(Annoy). Tree-based methods recursively partition the vector space into leaf nodes, each containing candidate vectors. Annoy constructs these partitions by spliting along hyperplanes defines by pairs of randomly selected vectors, and repeats this process to build a random forest of ntreesn_{trees}ntrees​ trees. At query time, it traverses each tree down the nearest leaf and sends the candidate vectors from all visited leaves to verification, where we interate PANORAMA.
   Locality-based(MRPT). MRPT also uses a forest of random trees, like Anooy does, yet split vis median thresholds on random linear projections rather than vis hyperplanes. This design ties MRPT to Johnson-Lindernstrauss guarantees, enbling recall tuning, and incorporates voting across trees to filter candidates. We integrate PANORAMA as-in the refinement pahse.
