VSAG, An Optimized Search Framework for Graph-based ANNS

Overview

  Recent graph-based ANNS algorithms have achieved high search accuracy with pratical efficiency. Despite the advancements, these algorithms still fac performance bottlenecks in production, due to the random memory access patterns of graph-based search and the high computational overheads of vector distances. In addition, the performance of a graph-based ANNS algorithm is highly sensitive to parameters, while selecting the optimal parameters is cost-prohibitive, e.g., manual tuning requires repeatedly re-building the index.
  This paper introduces VSAG, an open-source framework that aims to ehance the in production performance of graph-based ANNS algorithms. VSAG incorporates three key optimizations:

  1. efficient meory access: it reduces L3 cache misses with pre-fectching and cache-friendly vector organization
  2. automated parameter tuning: it automatically selects performance-optimal parameters without requiring index rebuilding
  3. efficient distance computation: it leverages modern hardware, scalar quantization, and smartly witches to low-precision representation to dramatically reduce the distance computation costs

Details

Framework

  VSAG integrates memory access optimization, automatic parameter tuning and distance computation optimizations into different phases for the search process. First, the PRS manages the storage of vector data and graph indexes. It stored low-precision quantized codes of vectors alongside high-precision orignal vectors to support distance computation optimization. This redundant storage is also utilized to balance resource usage and reduce memory access costs. Second, during each hop of the search process when exploring a vector, we employ Deterministic Access Greedy Search to accelerate memory access by minimizing cache misses. Finally, the visited results are pushed into a heap. We apply Selective Re-Rank to combine low- and high- precision distances, ensuring effective search performance. The search process then pops the nearest unfinished point from the heap and proceeds to the next hop. Throughout the entire life-cycle of the index, VSAG uses a smart auto-tuner to select parameters that deliver the best search performance across varying environments and retrieval requirements.

Memory Access Optimization

  deterministic Access: Distance computations for neighboring vectors often incur random memory access patterns in graph-based algorithms, leading to significant cache misses. VSAG addresses this by integrating software prefetching through a Deterministic Access strategy. VSAG strategically inserts prefetch instructions during and before critical computations, proactively loading target data into L3/high-level caches. Further more, the VSAG framework efficiently mitigates suboptimal prefetch operations through batch processing and reordering of the access sequence.
  PRS: VSAG introduces a Paritcal Redundant Storage(PRS) design(embeds compressed neighbor vectors at each graph node), which provides a flexible and high-performance stirage foundation to optimize both distance computations and memory access while balancing storage and computational resource usuage
  Balance of Resources: In particular, the system uses a parameter called the redundancy ratio δ\deltaδ to control the Balance of Computational Efficiency and Memory Utilization. In compute-bound scenarios with high-throughout demands, VASG adaptively increases the redundancy ratio to mitigate cache contention, thus minimizing CPU idle cycles during memory access while preserving storage efficiency.

Automatic Parameter Tuning

  VSAG address papramter selection complexity through a tripartite classification system with specialized optimization strategies:

  • Environment-level Parameters(ELPs): VSAG employs a grid search to identify optimal configurations for peak QPS performance through systematic parameter space exploration
  • Query-level Parameters(QLPs): VSAG implements multi-granular tuning strategies, including fine-grained adaptive optimization that dynamically adjusts parameters based on real-time query difficulty assessments
  • Index-level Parameters(ILPs): VSAG introduces a novel mask-based index compression technique that encodes multiple parameter configurations into an unified index structure. During searches edge-label filtering dynamically emulates various construction parameters, thereby reducing index-level parameters to query-level equivalents while keeping a single physical index

Distance Computation Optimization

  Quantization methods can efficiently accelerate distance computation. However, naive quantization approaches often result in significant accuracy degradation. VSAG uses Selective Re-rank to improve efficiency without sacrificing the search accuracy. Furthermore, specific distance metrics(i.e., euclidean distance) can be strategically decomposed and precomputed, effectively reducing the number of required instructions during actual search operations.

Memory Access Optimzation

  The fundamental strategy for mitigating cache-related latency lies in effectively utilizing vector computation intervals to prefetch upcoming memory requests into cache. In VSAG, three primary optimization strategies emerge for maximizing cache utiliation efficiency:

  • Leveraging software prefetching to improve cache hits rates
  • Optimizing search patterns to enhance the effectiveness of software prefetching
  • Optimizing the memory layout of indexes to efficiently utilize hardware prefetching

Software-based Prefetch: Making Random Meory Accesses Logically Continuous

  As shown in Figure 2, when computing vector distances, the vector is loaded sequentially from a segment of memory. But modern CPUs support software prefetch instructions, which can asynchronously load data into different levels of the cache. By leveragign the prefetch instruction to preload data, we can achieve a nrea-sequential memory access pattern from the CPU level. This indicates that data is preloaded into the cache before the CPU requires it, thereby preventing disruptions in the computational flaw caused by random memory address loads. More specifically, prior to computing the distance for the current neighbor, the vector for the next neighbor can be prefeched. As detailed in Figure 2, the green flow represents the use of prefetching. From the perspective of the CPU, the majority of distance computations make use data that has already been cached. Furthermore, because prefetching operates asynchronously, it does not obstruct ongoing computations. The synergy of asynchronous prefetching and immediate data access optimizes the utilization of CPU computational resources, thereby substantially enhancing search performance.

  The software-based prefetch mechanism initiates the fetching of next-neighbor vector upon completion of each neighbor computation. However, this method results in redundant operations because previously visited neighbors that do not require distance computations still generate prefetch time cost. In this section, we present two dedicated strategies to address the challenges.
  Deterministic Access: In contrast to prefetching during edge access checks, VSAG exclusively prefetches only those edges that have not been accessed. The mechanism begins by batch-processing all neighbor nodes to determine their access status. Following this, unvisited neighbors are logically grouped, and prefetching is performed collectively. This strategic approach ensures that each prefetched memory address corresponds exclusively to computation-essential data, thereby enhancing prefetching efficiency and minimizing redundant memory operations.
  Stride Prefetch: Batch processing ensures that each prefetch retrieves data intended for future use. However, prefetch effectiveness varies due to the asynchronous nature of prefetching and the absence of a callback mechanism to confirm prefetch completion. Optimal performance occurs when the required data reside in th cache precisely when the computation flow demands it. Premature prefetching risks cache evicion, while delayed prefetching negates performance again. The necessitates balancing prefetch timing with computation duration. To address this, stride prefetching dynamically aligns hardware computation throughput with software prefetching rates, maximizing prefetch utility. The key parameter, the prefetch stride w\mathcal{w}w, determines how many computation steps occur before each prefetch.

PRS: Flexible Storage Layout Boosting Search Performance

  While incorporating well-designed prefetch pattern into search processes can theoretically improve performance, their inherent limitations of software-based prefetch compared with hardware-based prefetch prevent guranteed memory availability for all required vectors(i.e., advisory but nor mandatory commands, cache line contention, intrinsic cost disparity between prefetch mechanisms).
  Hardware-based prefetching relies on hardware mechanisms that adaptively learn from cache miss events to predicate memory access patterns. The system employs a training buffer that dynamically identifies recurring data access sequences to automatically prefetch anticipated data into the cache hierarchy. The training mechanism shows particular effectiveness for Sequential Memory Access patters, where it can rapidly detect and exploit sequential memory access characteristics(IVF 系列索引比较适合). Conversely, graph-based ndexing architectures exhibit irregular access pattern with poor spatical locality, resulting in inefficient Random Memory Access. The inherent randomness of access sequences prevents the training buffer from establishing effective pattern recognition models.
   VSAG integrates the benefits of space-partitioned indexes into graph-based indexing algorithms through redundant vector torage. By co-locating neighbor lists with their corresponding vectors within each node’s data structure, it achieves Sequential Memory Access. This design ensures that neighbor retrieval operations only require sequential access within contiguous memory regions, thereby fully leveraging hardware prefetching capabilities.[ Figure 1]
  To address the computational-memory resource imbalance caused by fixed instance specifications in industrial applications, we propose PRS to take advantage of hardware prefetching to reduce CPU idle time. A dynamically tunable redundancy ratio δ\deltaδ controls the proportion of redundantly stored neighbor vectors in graph indexes, balancing prefetch efficiency and CPU utilization. When δ=1\delta = 1δ=1, full redundancy maximizes hardware prefetch benefits, achieving peak CPU utilization at the cost of higher memry consumption. In constrast, δ=0\delta = 0δ=0 eliminates redundancy to minimize memory usage but sacrifices prefetch efficiency.

Automatic Parameter Tuning

Search-based Automatic ELPs Tuner

  The environment-sensitive parameters(i.e., the prefetch stride w\mathcal{w}w, the prefetch depth vvv) xclusively influence algorithm efficiency metrics(e.g., QPS) while maintaining consistent effectiveness outcomes(e.g., Recall), thereby enbling independent optimization distinct from core algorithmic logic. The VSAG tunning framework implements an optimizaed three-step procedure:

  1. Conduct an exhaustive grid search for all combiniations of environment-dependent parameter
  2. Evaluate performance metrics using sampled base vectors
  3. Select parameter configurations that maximize retrieval speed while maintaining operational stability

Fine-Grained Automatic QLPs Tuner

  Experiments results show that assigning personalized optimal retrieval parameters to each query can improve performance by 3-5 times. To address the observation of QLPs, we propose a Decision Model for query difficulty classification, enabling personalized parameter tuning while maintaining computational efficiency. We introduce a learning-based adaptive parameter selection framework through a GBDT classifier with early termination capabilities, demonstrating superior performance in fixed-recall scenarios. The model architecture addresses two critical constraints: discriminative power and computational efficiency. Our feature engineering process yields the following optimal feature set:

  • Cardinality of scanned points
  • Distribution of distances among ccurrent top-5 candidates
  • Temporal distance progression in recent top-5 results
  • Relative distance differentials between top-K candidates and optimal sulution

  The candidate size is initially set to a relatively large value(e.g., efs=300ef_{s}=300efs​=300). During the search process, after ttraversing certain hops in the graph, we employ the decision tree along with the current features to determine whether the candidate size should be reduced, which effectively prevents overly simple queries from undergoing extensive and unnecessary search process.

Distance Computation Acceleration

  Recent studies illustarte that the exact distance computation takes the majority of the time cost of graph-based ANNS. Approximate distance techniques, such as scalar quantization, can accelarate this process at the cost of reduced search accuracy. VSAG adopt a two-stage approach that first performs an approximation distance search followed by exact distance re-ranking.

Distance Computation Cost Analysis

  VSAG employs low-precision vectors during traversal operations while resercing precise distance computations exclusively for final result reranking. The dual-precision architecture effectively minimizes distance computation operations(DCO) overhead while preserving search accuracy through precision-aware hierarchical processing.
  It we only consider the cost incurred by distance computation, the total distance computation cost be expressed as follows:

cost=costlp+costhp=nlp⋅tlp+nhp⋅thpcost = cost_{lp} + cost_{hp} = n_{lp} \cdot t_{lp} + n_{hp} \cdot t_{hp} cost=costlp​+costhp​=nlp​⋅tlp​+nhp​⋅thp​

  The optimization of nlpn_{lp}nlp​ is closely related to specific algorithm workflow, while tlpt_{lp}tlp​ is primarily determined by the computational cost of FLOAT32 vector operations - both of which remain relatively constant. Consequently, the VSAG framework focus primarily on optimizing the parameter tlpt_{lp}tlp​ and nhpn_{hp}nhp​ in the three following ways:

  • The combination of quantization techiniques, hardware instruction set SIMD, and memory-efficient storage achieves exponential reduction in low-precision distance computation(tlpt_{lp}tlp​)
  • Enhanced quantization precision by parameter optimization mitigates candidate inflation from compression loss, achieving sublinear growth in required low-precision conputations(n_{hp})
  • Selective re-ranking with dynamic thresholding establishes an accuracy-efficiency equilibrium, restricing high-precision validation(nhpn_{hp}nhp​) to logarithmically scaled candidate subset

Minimizing Low-Precision Computation Overhead

  SIMD and Quantization Methods: Modern CPUs employs SIMD instruction sets(SSE/AVX/AVX512) to accelerate distance computations rhtough vectorized operations. The instructions process 128-bit, 256-bit, or 512-bit data chunks in parallel, with vector compression techniques enabling simultaneous processing of multiple vectors. Besides, Prodcut Quantization(PQ) enables high compression ratios for batch processing through SIMD-loaded lookup tables. While PR-Fast Scan excels in partition-based searches through block-wise computation, its effectiveness diminishes in graph-based searches due to random vector storage patterns and inability to filter visited nodes, resulting in wasted SIMD bandwidth. In contrast, Scalar Quantization(SQ) proves more suitable for graph algorithms by directly compressing vector dimensions(e.g., FLOAT32 -> INT8/INT4) without requiring lookup tables. As demonstrated in VSAG, SQ achieves the optimal balance between compression ratio and precission preservation while fully utilizing SIMD accelation capabilities, making it particully effective for memory-bound graph traversals/
  Distance Decomposition: VSAG optimizes Euclidean distance computations by decoupling static and dynamic components. The system precomputes and caches invariant vector norms during database indexing, then combines them with real-time dot product computations during queries. This decomposition reduces operational complexity while preserving mathematical equivalance, as shown by the reformulated distance:

This space-time tradeoff reduces the subtraction CPU instruction in distance computation, while saves one CPU cycle.

Improving Quantization Precision

  Scalar quantization(SQ) compresses floating-point vectors by mapping 32-bit float values to lower-bit integer representations in each dimension. In SQ-b(where b denotes bit-width), the dynamic range is uniformly partitioned into 2b2^{b}2b intervals. For SQ4(4-bit), this creates 16 intervals over [0,1][0,1][0,1], where the k-th interval corresponds to [(k−1)/15.k/15)[(k - 1)/15. k/15)[(k−1)/15.k/15). Values within each interval are encoded as their corresponding integer index. However, pratical implementations face critical range estimation challenges: direct use of observed min/max values proves subpotimal when data outliers existing.
  To enhance robustness, we propose Truncated Scalar Quantization using the 99th percentile statistics rather than absolute extremes. This approache discards outlier-induces distortions while preserving quantization resulution over the primary data distrbution, achieving superior balance between compression efficiency and numerical precision as shown in Firgure 6.

Selective Re-rank

  Quantization methos can significantly enhance retrieval efficiency, but quantization error may lead to substantial recall degradation. While re-ranking with full-precision vectors can mitigate this performance loss. Howevre, applying exhausitive re-ranking to all candidates is inefficient. The VSAG framework addresses this challenge through selective re-ranking, effectively compensating for approximation erros in distance computation without compromissing system performance. A straightforward approache is to select only the candidates with small low-precision distances for re-ranking. The optimal number of candidates requiring re-ranking varies significantly depending on query characteristics, quantization error distribution, and search requirement kkk. To address this dynamic requirement, VSAG implements DDC scheme that can automatically adapt re-ranking scope based on error-distance correlation analysis.
