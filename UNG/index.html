UNG, Navigating Lables and Vectors, A Unified Approach to Filtered Approximate Nearest Neighbor Search

Overview

  To tackle the challenges of FANNS, we introduce a novel approach called the Label Navigating Graph, which encodes the containment relationships of label sets for all vectors. Built upon graph-based ANNS methods, we develop a general framework termed Unified Navigating Graph(UNG) to bride the gap between label set containment and vector proximity relations:

  • We introduce the novel concept of the Label Navigating Graph, which serves as a dynamic guiding mechanism to strictly constrain the ANNS exploration within the filtered search space by leveraging the Entry Label Sets retrieved for the query labels
  • We propose a general UNG framework tailored for filtered ANNS, enabling seamless integration of various graph-based ANNS algorithms which gurantees versatility, fidelity, completemess and adaptability
  • By using Vamana as the underlying graph index, we conduct extensive experiments on 10 realworld datasets to compare the UNG framework with state-of-the-art fiiltered ANNS algorithm

Details

  UNG first partitions the dataset according to the label set fif_{i}fi​ associated with each vector viv_{i}vi​, such that the vectors sharing the identical label set fall into the same group. Then, UNG builds a derected acyclic graph named Label Navigating Graph to encode the label set containment relationships, where an edge from label set %f% to f′f^{'}f′ indicates that f′f^{'}f′ is the Minimum Superset of fff. To ebale efficient ANNS, for all vectors within the group of each lable set, UNG can integrate existing graph-based ANNS approaches to build a proximity graph. Then Cross-group Edges are added following the direction of edges in LNG, which bridges the gap between vectors from different graphs. It consistutes a Unified Navigating Graph G combining both the LNG and proximity graphs. Given a query vector vqv_{q}vq​ with label set fqf_{q}fq​, by retrieving the Entry Lable Sets which directly cover all lables in fqf_{q}fq​, a greddy search from the entry vectors with such entry label sets delivers the final nearest vectors for vqv_{q}vq​.

Dealing with Data update

  Note that UNG is general framework built on a graph-based ANNS algorithm. Thus, the underlying graph index update mechanism aplies when inserting or deleting vectors in a group VfV_{f}Vf​. Next, we dicuss how to update the trie tree, label navigating graph(LNG), and cross-group edges in the UNG index.
  Insertion: When adding a vector viv_{i}vi​, if its label set fif_{i}fi​ is new, we insert fif_{i}fi​ into the trie tree and update the LNG according to minmum superset relationships between all subsets and the minimum supersets of fif_{i}fi​. If fif_{i}fi​ already exists, there is no need to update the trie tree and the LNG. Then, for each in-neighboring label set f′f^{'}f′ of fif_{i}fi​ in the LNG, the cross0group edges will be rebuilt for all vectors in group Vf′V_{f^{'}}Vf′​. Finally, we create the cross-gruop edges that starts from viv_{i}vi​.
  Deletion: When removing a vector viv_{i}vi​, if no vectors remain with label set fif_{i}fi​, we remove fif_{i}fi​ from the trie tree and then update the LNG similarly. If fif_{i}fi​ is still used, the trietree and the LNG remain unchanged. Then, for each in-neighboring lebel set f′f^{'}f′ of fif_{i}fi​ in the previous LNG, we rebuild cross-group edges for vectors in the group Vf′V_{f^{'}}Vf′​.
