VBASE, Unifying Online Vector Similarity Search and Relational Queries vis Relaxed Monotonicity

Overview

  An increasing need for more sophisticated vector queries requires integrating vector search systems with relational databases. However, high-dimensional vector indices do exhibit monotonicity, a critical property of coventional indices.The lack of monotonicity forces existing vector systems to rely on monotonicity-preserving tentative indices, set up temporarily for a target vector’s TopK nearest neighbors, to faciliate queries.
  This paper preposes VBASE, a system that efficiently supports complex queries of both approximate similarity search and relational operators. VABSE identifies a common property, relaxed monotonicity, to unify two seemingling incompatible systems.[ 基于松弛一致性，将向量索引和标量索引以一种统一的方式集成到数据库中 ]

Details

Division Between Databases and Vector Search Systems

  Vector search and database systems differ in their ways of using index, a critical structure to speed up queries. An important property of conventional indices like B-tree is monotonicity. This property ensures that a query can traverse the data-set guided by an index monotonically along a certain directrion. This often avoids total data scan, therefore enables efficient query execution.
  However, it is prohibitively expensive for high-dimensional vector indices to preserve monotonicity, because of the curse of dimensionality. Instead, they are often organized as a graph or cluster-based irregular structure, which follows monotonicity approximately. Traversing such vector indices does not gurantee a strict monotonic order in terms of distances to a target vector, but it enbales a system to efficiently determine when it’s unlikely for new traversals to reach closer vectors to a target than the current K ones.

VABSE Design

Relaxed Monotonicity

  There is a common two-pahse traversal pattern in most vector indices: In the first phase, the index tracersal approaches the target region approximately in spite of large oscillations in vector distances; In the second phase, the index traversal stabilizes and steadily departs from the target vector region in an approximate way. Thus a TopK search query could terminate early when it enters the second phase as further traversals are unlikely to identify more similar vectors.

  To determine whther it enters phase two, a query needs to understand RqR_qRq​, the radius of the neighbor sphere centered around qqq, and whether MqsM_{q}^{s}Mqs​, the distance between the query’s current index traversal position(denoted as traversal step sss) and qqq, is greater than RqR_qRq​, i.e., it is traversing beyond the neighborhood of qqq.

Unified Query Execution Engine

  With relaxed monotonicity, VBASE builds a unified query execution engine based on Volcano Model, which resues the traditional Open, Next, and Close interfaces to leverage index traversal so that no change is required for conventional indices[ 这里需要改造向量索引对外接口，使之可以支持 Next 操作，持续迭代对外输出搜索结果 ].
  VBASE also modifies the termination condition based on relaxed monotonicity. In paticular, VABSE extends the original termination condition with relaxed monotonicity check. In addition to the original query termination condition, VBASE performs relaxed monotonicity check by inspecting Eq3(query execution would only stop if both the original termincation condition and relaxed monotonicity check were passed)
  The unified engine enables VBASE to preserve full compatibility with traditional database and creates new optimization opportunities for vector queries. For example, instead of filtering after TopK, VABSE can perform filtering during index traversal with flexible termincation conditions. Moreover, the unified engine allows the incorporation of a refined NRA algorithm, which can significantly improve the performance of multi-column vector query

VBASE Implementation

  We implement a common relaxed monotonicity check through two queues:(1) a prioroty queue with size EEE called smallestQueue, to keep the visited nearest neighbors of a target vector qqq during the traversal; (2) recentQueue of size www to track the most recent traversal window. When a new vector vvv is visited via index traversal, smallestQueue and recentQueue are updated accordingly. And the relaxed monotonicity is performed by calculating the current traversal state according to Eq3.
