EIS, Elastic Index Select for Label-Hybrid Search in Vector Database

Overview

  Real-world vector embeddings are usually associated with extra labels, such as attributes and keywords and many applications require the nearest neighbor search that contains speecifix lables. A straightforward approache is to materialize all possible indices according to the complete query label workload. In this paper, we leverage the inclusion relationships among query label sets to construct partial indexes, enabling index sharing across queries for improved construction efficiency.

Details

Related Work

  Various ways of AKNN indexing support attribute value filtering, where the attribute can be a label, a keyword, or a numerical value. We divided them into two classes:

  • Attribute-Free Approach: These methods do not require attributes during index construction and determine whether a vector is filtered on the fly. Based on this, the Prefiltering and PostFiltering search have been proposed and many systems(i.e. ADB,VBASE,Miluvs) select different srategies based on the cost model. Since these strategies completely ignore the attributes during index construction, the search accuracy and efficiency have a large gap compared to the method that involves attributes.
  • Attribute-Involved Approach: These methods are constructed based on different attributes:
    • For range filter queries on numerical attributes, recent methods construct a segment tree to construct the query interval. I.e., SeRF uses the idea of compression to compress the index corresponding to each query range interval into a compact one
    • For label attributes, UNG uses the inclusion relationship of the label set and constructs cross-group edges to ensure that the graph index only searches the filtered vectors
    • Beyond the above methods, NHQ takes the attribute as part of the vector similaity computation, which requires manually adjusting the weight of the two parts. The recent approach DEG solves this by adjusting the graph neighbor dynamically.

Extend with Limited Space

  A nattural question is how to select the index to achieve maximum efficiency under a space limitation. Since we use the elastic factor to model the search efficiency, we transfer the problem to maximizing the elastic factor bound with a given space limitation. Formally, we aim to select a subset I′\mathbb{I}^{'}I′ from all possible index set I\mathbb{I}I such that the elastic factor e(S(Lq),I′)e(S(L_q), \mathbb{I}^{'})e(S(Lq​),I′) bound is maximum for given label-hybrid queries Lq∈LL_q \in \mathcal{L}Lq​∈L and the cost of selected index set I′\mathbb{I}^{'}I′ is less than a threshold τ\tauτ (Fixed Space Index Selection(SIS) problem).
  Note that SIS is also a NP-hardness problem, we reuse the greedy selection method for EIS to solve the problem. We still use binary search for the best elastic bound ccc. We update the result if the greedy solution is lower than the threshold τ\tauτ with a better elastic factor bound. This requires O(log)O(log)O(log) calls to the greedy algorithm. In practice, binary search with the greedy method takes up less than 11%1 of the construction time overhead, ususally 1−21-21−2 seconds for even large L\mathcal{L}L.
