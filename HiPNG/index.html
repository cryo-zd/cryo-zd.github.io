HiPNG, Efficient Interval-Filtering ANNS via Hierarchical Interval Partition Navigating Graph

Summary

   In this paper, we introduce interval-filtering ANNS(IF-ANNS), a novel yet general retrival task where both base and query vectors are associated with numerical intervals. The goal is to retrieve nearest neighbors whose intervals are fully contained within the query interval.
   To efficiently address this problem, we propose Hierarchical Interval Partitioning Navigating Graph(Hi-PNG), a framework that hierarchically partitions the interval space to efficiently locate the search space, eliminating unnecessary computations and achieving significant speedups. In summary, we make the following contributions:

  • We formulate the IF-ANNS, a more general constraint work that generalizes existing filtered ANNS and Rf-ANNS problems. IF-ANNS adds interval constraints for base and query vectors, enhancing support for complex nearest neighbor search.
  • We develop the Hierarchical Interval Partition Navigating Graph(IF-ANNS). By embedding proximity graphs into tree nodes, it efficiently partitions the interval space, balancing query performance and memory efficiency for IF-ANNS.
  • We provide a thretical analysis of the search process and estimiate the candidate space reduction. Under unifor interval distribution, the candidate space can be reduced by a factor of 6p2\frac {6} {p^{2}}p26​(where p<1p \lt 1p<1) compared to full space

Hierarchical Interval Space Partition

   The candidate space includes all results that satisfy the query constraints. However, it may be only a small part of the interval space, and searching the full interval space is both costly and unnecessary. To address this, we employ a hierarchical interval space partitioning to efficiently locate the candidate space.
   Definition 3.6(Hierarchical Interval Space Partition). The interval space Ω\OmegaΩ is recursively partitioned into rectangle regions, referred to as fields. Starting with the entire interval space F0F_{0}F0​, the partitioning process follows these rules: (1) Each field FXF_{X}FX​ is divided into four mutually exclusive child fields, denoted as F4x+1,F4x+2,F4x+3,F4x+4F_{4x+1}, F_{4x+2}, F_{4x+3}, F_{4x+4}F4x+1​,F4x+2​,F4x+3​,F4x+4​, such that Fx=⋃i=14F4x+i,F4x+i∩F4x+j=∅ for i≠j\mathcal{F}_x = \bigcup_{i=1}^{4} \mathcal{F}_{4x + i}, \quad \mathcal{F}_{4x + i} \cap \mathcal{F}_{4x + j} = \emptyset \text{ for } i \neq jFx​=⋃i=14​F4x+i​,F4x+i​∩F4x+j​=∅ for i=j. (2) The partitioning continues recursively until the number of intervals within each field is less than or equal to a predefined threshold, call leaf size lslsls. This recursive process results in a hierarchical structure of fields of varying sizes, facilitating efficeient searchs by focusing on smaller regions.

   Observation. Given any two fields FiF_{i}Fi​ and FjF_{j}Fj​, if FiF_{i}Fi​ contains FjF_{j}Fj​, then all sub-fields within FjF_{j}Fj​ will be contained by the field FiF_{i}Fi​. Through fine-grained heirarchical interval space partitioning, we can quickly find the region meeting query interval constraints and reduce the candidate space.

Partition Point Selection

   This section investigates how to select a partition point in the interval space during partitioning.

  1. Interval Space Balanced Partitioning. Based on the principle of interval space balancing, we select a partition point to ensure an even division of the field. To achieve this, we identify the center point of the field, which lies in the midpoint of the diagonal connecting the upper left and lower right corners of the rectangle. By this, the field is evenly divided into four sub-fields, each maintaining a balanced area in the interval universe.
