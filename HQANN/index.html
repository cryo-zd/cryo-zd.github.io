HQANN, Efficient and Robust Similarity Search for Hybrid Queries with Structured and Unstructured Constraints

Summary

   The in-memory ANNS algorithms have achieved great success for high-recall query processing, but are extremely inefficient when handling hybrid queries with unstructured constraints. In this paper, we present HQANN, a simple yet highly efficient query processing framework which can be easily embedded into existing proximity graph-based ANNS algorithms. We guarantee both low latency and high recall by leveraing navigation sense among attributes and fusing vector similarity search with attribute filtering.
   In HQANN, we argue that attibutes should be in the leading position for proximity graph-based hybrid query processing. And following the fusion strategy in Figure 1, we merge the search and filtering stage by going through the composite graph, leading to high recall hybrid ANNS algorithms with negligible extra computation overhead. The maincontributions of this paper are as follows:

  1. We introduce a novel hybrid query processing gramework, HQANN, which tactfully manages hybrid queries in a fused way. HQANN is generally applicable to many proximity graph-based ANNS algorithms. We demonstrate the flow of construction and search for proximity graphs can be easily adapted to the proposed framework
  2. We propose to leverage the navigation sense among attributes to traverse the proximity graphs.

Details

HQANN

   The performance of hybrid query processing is largely determined by the efficiency of setting attributes, as they determine the accurate search space for future vectors. From this motivation, we propose HQANN. In HQANN, all attributes is treated as a native part of the fusion distance metric, which can make full use of the inhereny Navigation feature of proximity graph-based ANNS solutions.

Fusion Distance Metric

   The key point for hybrid query processing is to define an effective distance metric to facilitate analytics on attributes. Give the fused distance function as the basic premise, a composite proximity graph-based index can be constructed and retrieved without invading its original execution flow.
   Generally, performance of a proximity graph is largely determined by the quality of the graph node neighbors. Since the feature vector search space of hybrid queries is determined by the attributes, we treat attributes as the dominant part of the fusion distance metric, which means a graoh node tend to establish links with datapoints containing the same or very similar attributes. In such circumstances, a hybrid query can quickly locate those graph nodes with exactly the same attributes and conduct detailed vector similarity search among their neighborhoods.

Composite Proximity Graph

   The basic idea od HQANN is that we obtain a fusion distance metric by fine-tuning the attribute distances via feature vector distances. More specifically, the distance between two hybrid datapoints is mainly defined by their attribute distance, and their future vector distance serves as adjustment factor with limited impact on the fusion distance. During the construction stage of the proximity graphs, datapoints with the same or very similar attributes will linked first as they tend to have much smaller distcne compared with our candidates. The the remaining neighborhood vacancies would be filled up with datapoints that are relatively distant in attributes, which strongly maintains the connectivity of the graph. In the search stage, a hybrid query traverses the graph until it reaches the first node with the same attribute, by fully scanning the node’s neighborhoods, a high quality return candidate list can be efficiently accessed.
   The overall configuration of the composite proximity graph is shown in Figure 2. With attribute plays a major role in the fusion distance metric, we actually accomplish the filtering-search strategy by simple traversal of the composite graph. The numerical differences among attribute distances and feature vector distances skillfully exploiting the proximity graphs’ essence of voracious searching for graph nodes closest to the query. Hence, HQANN is robust against scale of attributes and remains highly efficient.
