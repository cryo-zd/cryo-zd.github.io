SHG, Acceleraing ANNS in Hierarchical Graphs, Efficient Level Navigation with Shortcuts

Overview

   Hierarchical methods still face efficicency bottlenecks because they rely on exhaustive and level-by-level traversals within hierarchical graphs. This paper introduces SHG, a novel hierarchical graph-based index that enhances search efficiency by bypassing intermediate and redundant levels. Specifically, SHG leverages a hierarchical vector compression method to reduce the time spent on distance computations, and employs a new data structure called shortcuts to determine the number of intermediate levels that can be safely skipped.

Shortcut-Enabled K-ANN Index

Key Component of Our Index: Shortcut

   The primary role of shortcut is to infer the number of levels that can be skipped when traversing the hierarchical graphs. This allows the shortcut to serve as an auxililary data struxture that complements ANN inexes and significantly improves the query efficiency.(In this section, we use the HNSW as an example to introduce out solution).
   Definition (Shortcut). Given a query vector and an entry vector ep in the hierarchical graph at a vertain level lll, the shortcut leverages the (approximate) distance dis^\hat{dis}dis^ between q and eq to infer the number of levels, denoted by f(dis^)f(\hat{dis})f(dis^), that can be skipped from lll. The shortcut also gurantees that ep remains the entry vector of the hierarchical graph at the target level with probability.

Index Construction

   Although piecewise linear models have achieved significant success in building learned indexes, we still face two major challenges to build learned shortcuts:

  • Determining Safe Skip Levels. There is currently no guideline for deciding the number of levels that can be safely skipped. Prior work on ANN search has not considered building shortcuts in hierarchical graphs, making this a novel challenge.
  • Generating Sufficient Training Samples. Unlike existing studies on learned indexes, the training data for our model is not explictly provided by the input dataset. Therefore, we need to devise an effective algorithm to generate sufficient training samples to build an accurate learned shortcut.

   Guideline for Skipping Levels. We propose a density-based criterion to determine how many levels can be safely skipped. Specically, consider ooo as the nearest neighbor of the query vector qqq at level xxx, which will serve as the entry point at level x+1x + 1x+1. If there are already numerous nearby vectors at level xxx that exhibit short distance to qqq(but longer than ooo), it is likely that ooo will remain the nearest neighbor to qqq at level x+1x + 1x+1. In such cases, traversing the proximity graph from ooo at level x+1x+ 1x+1 becomes redundant. Since ooo is the nearest vector to qqq, the nearby vector of qqq should also be close to ooo. k-ANN Search

   Our index accelerates search efficiency in two ways:

  • Using the vector compression method aforementioned, upper levels require fewer dimensiona to compute approximate distances. This significantly reduces computational costs
  • Using the learned shortcut, we can infer the number of levels that are confident enough to be skipped. This reduces the time spent traversing unncessary levels.

Index Maintenance

   Although ANN search is often studied over static data, some recent studies also consider the issue of index maintenance over dynamic updates. In general, dynamic updates involve two scenarios: deleting existing vectors and inserting new vectors. Our index SHG can effectively handle either scenario with the following strategies:

  • Deletion. In hierarchical graph based indexes, such as those implemented in the HNSW library, a prevalent method for deleting vectors is to mark them as deleted and verify these markers duting graph traversal. This method is also generally effective for our index. However, if a substantial portion of the dataset(e.g., 50%) has been deleted, up to half of the vectors each level may be non-existent. In such cases, search efficiency might be reduced, so it may be beneficial to rebuild the entrie index from scratch.
  • Insertion. Our index takes two main steps to insert a new vector: (i)Inserting in Hierarchical Graph: This steps adheres to the insertion routine for hierarchical graph-based indexes. Taking HNSW as an example, inserting a new vector involves establishing new edges between existing vectors and the new vector. It takes O(logN)O(log N)O(logN) time to insert a vector into a set of nnn vectors. (ii) Insertion in Learned Shortcut. Our SHG additionally performs insertions in learned shortcuts. In inserts the training data associated with the new vector and leverages the foundation model, PGM index, to process updates in O(logN)O(logN)O(logN) time.
