DEG, Efficient Hybrid Vector Search Using the Dynamic Edge Navigation Graph

Overview

  Bimodal data, such as image-text pairs, has become increasingly prevalent in the digital era. The Hybrid Vector Query(HVQ) is an efficient approach for querying such data. In this paper we propose a novel ANNS index, DEG, for the hybrid vector query probelm, which comprises three novel components: the greedy Pareto frontier search algorithm, the dynamic edge pruning strategy, and the edge seed method.

DEG Method

  The key lies in how to construct a graph-based ANNS index that is capable of handling varying α\alphaα values, which represents three challenges:

  1. How to compute the candidate neighbor set for each node: as α\alphaα changes, the distances bwteeen objects would change, and the appropriximate nearest neighbors of each node may vary significantly
  2. How to determine edges from the candidate set: for the HVQ problem, the α\alphaα value varies at query time, making the edge lengths dynamic, and the RNG pruning strategy does not work
  3. How to select the seed for the graph index in the HVQ problem: the start node(seed) of the graph impacts the search path length, which reflects the search efficiency. But the center now can vary significantly with different α\alphaα values[ 不建议像HNSW或NSG那样从固定或随机的起点(全图中心点)出发]

Candidate Neighbor Acquisition

  To tackle the first challenge, we propose rge GPS algorithm, whose high-level idea can be summarized in two points:

  1. Find the nearest beighbor of a node at any α\alphaα value is equaivalent to solving the problem of minimizing the distance function(Equation 1) at that α\alphaα value. But finding solutions for each possible α∈[0,1]\alpha \in [0,1]α∈[0,1] is unfeasible. Then we treat it as a multi-pbject optimzation problem with each vector distance as an object. To solve this, we compute the Pareto frontier as the solution
  2. Finding the extract Pareto frontier is expensive. Therefore, the GPS algorithm aims to find the approximate Pareto frontier to reduce index construction costs. The core idea of the GPS algorithm is that a neighbor’s neighbor is likely to be neighbor. By iteratively exploring neighbors of neighbors, it continuously optimizes the approximate Pareto frontiers.

Dynamic Edge Pruning Strategy

  To address the second challenge of determing edges from the candidate set, we propose a dynamic edge pruning strategy whose key idea is as follows:

  1. We assign an active range to each edge, covering the suitable α\alphaα values for that edge, within which that edge will not be pruned by the RNG’s property
  2. At query time, we use an edge for routing only if the query’s α\alphaα value intersects with its active range; otherwise we ignore it

Edge Seed Acquisition

  To address the fird challenge, we propose the a new edge seed method, which uses nodes that are farthest from the center for varying α\alphaα values as seed. Comparing tp using multiple approximate centroids, edge seed ensures that they are far from each other. In greedy search, only edge nodes close to the query are activated, while others are ignored due to their distance.
  To obtain the edge seed, we first calculate the centroid ccc of dtataset DDD, where c.e=avg(x.e)c.e=avg(x.e)c.e=avg(x.e) and c.s=avg(x.s)c.s=avg(x.s)c.s=avg(x.s) for all x∈Dx\in Dx∈D. Then we maintain the inverse Pareto frontier of the centroid ccc, which cosists of nodes that do not dominate any other nodes based on the distance from the centroid. Finally, we will update the edge seed set when inserting new node if it can be added to the inverse Pareto frontier of the centroid

Search Algorithm

  The seach algorithm of DEG is based on the greedy seach algorithm but with tow modifications:

  1. It dynamically skips edges whose active range do not interset with the query’s α\alphaα value
  2. It introduces an eraly stopping mechanism, that is, when calculating te hybrid distance, it first computes one of theindividual vector distances and compare it to the threshold distance in the greedy seach algorithm to determine whether to skip this node early

  Specifically, the algorithm first initializes an empty min-heap SSS as the candidate set, an empty max-heap RRR as the result set. Next, the seeds are added to SSS, RRR. In each iteration, the object with the smallest distance in SSS is fetched. If its distance to the query is larger than the maximum distance in RRR, the loop breaks, and the top 𝑘𝑘k objects in RRR are returned. Otherwise, its neighbors are checked, with some being skipped based on the active ranges and the query α\alphaα value. If the distance of a neighbor to the query is smaller than the maximum distance in RRR, the neighbor is pushed into SSS and RRR. Here, the early termination mechanism is used for acceleration. Future Directions
